sigfit - fitting mutational signatures to SNP data

Introduction

sigfit is used to estimate the proportions in which a set of mutational signatures are acting on a genome. It requires two inputs:

  • a vector of mutation counts - the number of times mutations of particular types (typically the 96 different tri-nucleotide contexts) occur in the genome

  • a matrix of mutational signatures, which describes the estimated probabilities of each mutational category being observed

Installation

sigfit is an R package. As it is in early development it is not on CRAN, but can be installed from github using devtools.

devtools::install_github("kgori/sigfit")

Usage guide

Generating data

This example will use the mutational signatures from COSMIC to generate simulated mutation counts, and then use sigfit to fit the signatures back to the simulated data.

First of all we need some mutational signatures to fit to our data. This command will fetch mutational signatures from COSMIC.

sigs <- sigfit::fetch_cosmic_data()

Letâ€™s use these signatures to simulate some mutation data. This code will generate 20000 mutations from a 4:3:2:1 mixture of signatures 1, 5, 7 and 11.

set.seed(1)
probs <- as.matrix(sigs[, c(1, 5, 7, 11)]) %*% c(0.4, 0.3, 0.2, 0.1)
mutations <- as.vector(rmultinom(1, 20000, probs))
names(mutations) <- rownames(sigs)

Here is what our simulated counts look like:

barplot(mutations, cex.names = 0.4, las = 2)

Fitting signatures

Now we can estimate the exposure of the data to each signature (pretending we donâ€™t already know that it was generated from 1, 5, 7, 11). sigfit uses Stan to run a Bayesian model that produces Markov Chain Monte Carlo samples. rstan::sampling arguments, such as iter, warmup, etc., can be passed through.

set.seed(1)
mcmc_samples <- sigfit::run_sampling(counts = mutations, signatures = sigs,
                                       iter = 1500, warmup = 500, seed = 1)

We can look at the fitted spectrum using plot_spectrum. This functions can pass through arguments to barplot 

sigfit::plot_spectrum(mcmc_samples, names.arg = names(mutations),
                        cex.names = 0.4, las = 2)

plot_exposures produces a barplot of the inferred exposure of the sample to each signature.

sigfit::plot_exposures(mcmc_samples, prob = 0.9, thresh = 0.01,
                         cex.names = 0.7, las = 2)

The bars in this plot are coloured blue if the estimated exposure value is â€˜sufficiently non-zeroâ€™ - it is difficult for the model to make hard assignments of which signatures are present or absent due to the non-negative constraint on the estimate, which means that the range of values in the sample will not include zero. In practice, â€˜sufficiently non-zeroâ€™ means that the lower end of the Bayesian HPD credible interval (prob) is above a threshold value (thresh).

In this example, sigfit has identified the 4 signatures used to construct the sample.
